SoloPro - Improving Input Field Usability

SoloPro is a home-buying startup that gives its customers 3% back after purchasing a house. A crucial part of their product is their real estate agent base, who all must register with SoloPro and define their geographic ranges of operation. This mini-project during my internship there focused on designing an input field that was flexible and easy to use.

In total, there were four iterations. The last version was
ultimately sent to the development team for implementation.

I made additional notes, in pink text. You'll see them in the image slides.

Version 1 and 2

I started off with a basic drop-down list input field for version 1. I quickly began revision for version 2. For version 2, I added an editable location list to help agents manage this information.

Version 3

In version 3, I changed the input method to a text box. The rationale behind this switch, although unconventional, was to allow for max flexibility in input format. Why?

Feedback from agents revealed that that agents define their areas of operation in many ways: some use zipcodes, some city-state, some cover entire states, etc. To reduce clutter, I decided that an intelligent, open-ended form able to suggest and verify inputs was best.

Cons: Users might be confused at first, so I added text and a green ? offering guidance.

Version 4

In this final version, I (1) expanded on the (?) guiding pop-up, (2) moved the instructional text into the text box, and (3) replaced the ledger-like location list with one that just lists out entries. Why?

  1. Better explanation at little visual cost; only shows when users click on the (?)
  2. Visually cleaner
  3. More comfortable to use an editable, simple list of past entries instead of a table full of missing column entries.

Reflections

  1. I would have like to run prototype tests before sending it off to development, to ensure its effectiveness. However, the company was pressed for time.
  2. Granted more time, I would have run usability tests after deployment to see how agents used the design, and decide on next steps.
